## 1. Release Summary

Version `0.1.0-alpha` establishes the foundational scaffolding for the KitchenOS platform. This release includes the multi-container Docker development environment, Python API routing paths, SQLite database configurations, and HTML5 / Tailwind CSS dashboard layout templates.

---

## 2. New Features & Modules

### 2.1. System Container Scaffolding
*   Docker Compose configuration launches backend API (Python modular server), development database (SQLite), and caching service (Redis) containers.
*   Central configuration management parses environment variables from configuration templates.

### 2.2. Interface Frameworks & Layouts
*   Responsive HTML5 dashboard shell layout with collapsibility sidebar navigation, styled using Tailwind CSS utility rules.
*   Central client entry points inside `frontend/js/app.js` using modular imports.

### 2.3. Routing & Status Indicators
*   API router base configured to route requests under versioned prefixes: `/api/v1/*`.
*   System health check endpoint deployed at `GET /api/v1/health`.

## 3. Deployment Requirements & Config

### Prerequisites:
*   Docker Desktop (v20.10+) and Docker Compose (v2.0+).
*   Port availability: 80 (Frontend web server), 8000 (Backend API).

### Installation Steps:
1.  Clone the repository.
2.  Copy `config/env.development` to the project root as `.env`.
3.  Launch containers: `docker compose up --build -d`.
4.  Apply SQLite schema setup: `docker compose exec backend python database/migration.py`.
5.  Access the web dashboard at `http://localhost`.
