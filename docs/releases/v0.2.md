## 1. Release Summary

Version `0.2.0-beta` introduces the core business modules of KitchenOS. This release includes the Recipe Management and Costing Engine, JWT-based authentication guards, real-time inventory level logs, and initial POS sale integrations.

---

## 2. New Features & Core Modules

### 2.1. Authentication & Security
*   JWT-based session authentication with token storage in secure cookies.
*   Role-based access control (RBAC) checked on both the frontend (DOM layout rendering guards) and backend (API validation checks).

### 2.2. Recipe Costing Engine
*   Recipe database support in SQLite with ingredient listing, portions definition, and prep instructions.
*   Dynamic recipe cost calculation based on active supplier catalogs.
*   Client-side recipe scaling to adjust quantities based on portion counts.

### 2.3. Inventory & Procurement
*   Real-time stock counts and inventory level logs with FIFO support.
*   Reorder threshold alerts displayed on the dashboard.
*   Automated purchase order generation.

### 2.4. POS Webhook Integrations
*   Webhook listener endpoint processes sales logs and deducts ingredients from inventory.

## 3. Database Migrations & Updates

This release includes database schema updates. Apply modifications by executing:
*   `python database/migration.py --upgrade` to add recipes, recipe ingredients, inventory levels, and purchase orders tables.

---

## 4. Deployment Scaffolding Config

### Copy Configuration:
Copy staging configurations: `cp config/env.staging .env`.

### Migrate & Seed Database:
```bash
docker compose exec backend python database/migration.py --upgrade
docker compose exec backend python scripts/seed-catalog.py
```

## 5. Known Issues & Workarounds

*   **Celery Redis Queues Connection Drop**: Background email delivery tasks can fail under intermittent network drops.
    *   *Workaround*: Celery auto-retry settings are enabled to repeat operations up to 3 times before logging exceptions.
